## Problem

No way to save Pi sessions to cly from within Pi, and no way to trigger a checkpoint (context file + summary + changelog) from a single command. The cly `agent-session ls` command required TTY (Bubble Tea picker), making it unusable from agents.

## Decisions

- **Created `extensions/modules/checkpoint.ts`** — the `/checkpoint` command. JS does: resolve session ID, find-or-create in cly via `cly agent-session save --json`, then emit prompt for LLM to write context/summary/changelog.
- **Created then removed `extensions/modules/save.ts`** — initially built a separate `/save` command, but it duplicated checkpoint logic. Removed.
- **Refactored cly `agent-session ls`** — outputs JSON by default (no TTY), added `--tui` flag for interactive picker, added `-d`/`--directory` flag, default provider changed from `claude` to `all`.
- **Added `--json` flag to cly `agent-session save`** — returns `{"name","id","provider","path","description","saved_at","created"}` so JS can parse it.
- **Session ID resolution** — uses `ctx.sessionManager.getSessionFile()` first, falls back to scanning `~/.pi/agent/sessions/` dir by mtime. Pure Node fs, no python dependency.
- **find-or-create pattern** — `findSessionInCly()` checks sessions.json for existing session ID. If found, returns name. If not, AI generates description, builds slug name, calls `cly agent-session save --json`.
